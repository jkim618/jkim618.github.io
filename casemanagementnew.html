Overview

Constituent Case Management involves handling cases related to citizens at the state and local government level. It includes the intake, processing, and recording of activities for cases, which can range from investigations to approvals. This workflow allows users to submit cases on behalf of citizens or for citizens to submit cases themselves.

I led the design of citizen portal landing page for Appian Constituent Case Management to enhance the user experience for citizens as well as creating the case creation process.

Role

Sole UX Designer

Duration

June 2022 - August 2022

Team

Product Manager, Engineers, QA

Problem

Feedback from stakeholders and partners highlighted a key challenge: the current citizen-facing experience lacks clear guidance. Upon navigating to the agency's welcome page, citizens may not immediately recognize the need to 'create a case' and what that exactly entails. This lack of clarity results in users not having enough context to take appropriate actions.

Solution

The redesign aims to provide a more guided user experience with clearer context. It includes suggested case types with explanations, a search feature for quick case identification, helpful documentation and resources, and the ability for users to create an account and register.

Design Goals

Based on the user needs and stakeholder requirements, I summarized the higher level goals I hope to achieve with the redesign.

    1. Enhance navigation: Improve the platform's navigation to allow users to move between sections seamlessly and access features efficiently.

    2. Clarify Information Hierarchy: Establish a clear and logical arrangement of content to help users easily find and understand relevant information.

    3. Create a Cohesive User Flow: Develop a consistent and intuitive flow throughout the dashboard to guide users through their tasks and interactions smoothly.

    4. Facilitate Data Exploration: Enable users to explore and analyze data effectively by providing robust filtering options, allowing them to refine their view and focus on specific information relevant to their needs.

Initial Wireframes

Following a session with stakeholders, I quickly created a preliminary wireframe to initiate discussions. Although it was a rough draft, it effectively communicated my ideas and highlighted the potential advantages and drawbacks.

Pros:

    1. Improved information hierarchy with a top-down approach.

    2. Ability to filter information based on loss drivers.

    3. Combined view of the solution matrix graph and table, providing comprehensive data in one glance.

Cons:

    1. Potential confusion from using two dropdowns for vertical and subvertical.

    2. The solution matrix graph appears small and cramped.

Key takeaways were to make the vertical and subvertical elements more prominent, enlarge the solution matrix graph, and defer vendor information display.

Solution Matrix Iteration

After receiving feedback from stakeholders, I created improved wireframes. I enlarged the solution matrix and replaced the vendor preview with a 'Compare Top Vendors' text link, using the freed space for solution information preview. However, both iterations had issues with the lengthy loss drivers column, even when using an expand-collapse format. To address this, I iterated again, turning the loss drivers into colored keys.

Design Decision 1: Dropdowns and Navigation Tabs for Data Selection

To address the limitations of relying solely on dropdowns for data selection, we implemented a dual approach. This includes a top-level filter using dropdowns and the introduction of navigation tabs. This enhancement provides users with more intuitive ways to filter and navigate through different datasets, improving overall usability and user experience.

Design Decision 2: Reorganizing Dashboard Sections for Improved Usability

I separated the "Loss Impact" and "Solution Matrix" sections into distinct pages in my later iteration, enhancing user experience by reducing cognitive overload. Users can now effortlessly switch between sections using dedicated buttons, promoting a more intuitive and streamlined experience. This reorganization not only improved platform performance by reducing the amount of content loaded at once but also enhanced content organization.

Design Decision 3: Use of Filter Chips

To provide users with more control over the displayed data, we implemented filter chips. These filter chips allow users to refine the displayed graph and table by selecting specific loss drivers.
