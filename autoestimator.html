Auto Premium Estimator

Streamlining auto premium calculations to enhance accuracy and clearly identify key drivers of premiums.

Role

Lead UX Designer

Project Context

June 2024 - Ongoing

Team: Engineers, Project Manager, QA

Tools

Figma, Figjam

Problem

The tool is designed to calculate insurance premiums for different vehicle categories, known as power units, and is tailored for insurance producers. However, the current tool presents several user experience challenges:

1. Disconnected Inputs: Users struggled to see how their inputs affected the table data, causing confusion and disrupting workflow.

2. Low Input Visibility: The input fields were unclear, making it difficult for users to understand how their entries influenced the premium estimates.

3. Cluttered Layout: The table required excessive horizontal scrolling, leading to information overload and making it hard for users to efficiently view and interpret the data.

Solution

To resolve these usability issues, I redesigned the tool with a more intuitive and user-friendly experience. The calculator was made interactive, providing real-time feedback as users adjusted inputs, allowing them to clearly understand how premiums were calculated. I also introduced specialized calculators for each vehicle category, reducing cognitive load and improving data interpretation.

Furthermore, I restructured the workflow to be vehicle-centric. Instead of the initial, counterintuitive process of adding locations first, users now begin by adding vehicle groups, followed by mapping them to locations. This logical progression simplified navigation and made the tool more aligned with user expectations.

Initial Ideas

To engage stakeholders, I developed two low-fidelity iterations addressing key usability issues. Both versions prioritized interactivity, allowing users to break down each factor and view details by zip code or location. This enhancement clarified how different inputs influenced calculations, improving user comprehension of location-specific impacts.

I also restructured the table to be vehicle-centric, enabling users to add vehicles first before mapping them to locations. This change aligned with the tool’s focus on auto premiums and corrected the previous design’s flawed logic.

Research and Insight

I presented the initial ideas and low-fidelity wireframes to the stakeholders, who responded positively, particularly valuing the detailed breakdown of the calculator. They did, however, provide additional requirements that necessitated further refinement of the design.

Key new requirements included integrating global inputs and creating context-specific calculations for individual vehicle groups. I returned to the drawing board to incorporate their feedback and enhance the design accordingly.

01. Global Inputs

Integrate inputs affecting the entire calculator, not just specific locations.

02. Context-Specific Calculations

Develop a calculator that displays relevant calculations for individual vehicle groups.

User Workflow

I developed a comprehensive user flow to address the new requirements and lay a strong foundation for the design. This flow captures all essential elements and guides the implementation of the new features. To finalize it, I facilitated a workshop with stakeholders, allowing us to collaboratively review and refine the user flow.
